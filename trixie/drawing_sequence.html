A brief tour of how Trixie's compositor. This is a guide for the director, and must NOT be used as a source for Trixie's compositor development

Trixie borrows windowing conventions from Blender, and follows a similar albeit a bit simplified pipeline

Trixie also uses a window-carving mechanism to assign spaces. Essentially the main window space is "carved" down to make space for now widgets

Editor
An editor context, typically subdivisible into a new window.
↓
Region
A context container. It packs both drawing and stateful data about the drawing context for the editor childs
↓
Box
A visual representation struct abuot _where_ the drawing space is and how big it is
↓
Viewport
A complex class that handles viewing space. It represents the view space in World-Space, Screen-Space and keeps track of world bounds.
* Viewport is queried by visual controllers to know about their required state
* Viewport is controlled by controllers to move the view space around canvases
* Viewport also works as a space transformer. It takes geometry in and spits transformed geometry out. It is used by renderer pipelines to squish worlds into the viewspace
* Viewport also takes Box-space input and spits out world-space input.

Input needs to pass by the Box through the viewport before reaching input handlers for any particular controller
